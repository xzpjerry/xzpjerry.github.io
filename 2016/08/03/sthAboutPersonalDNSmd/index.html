基础轮子，自用dns搭建与使用

发表于 2016-08-03   |   分类于 vps   |   热度  °C   |   评论

母鸡配置（借助dnsmasq实现）：

  • 安装
    yum install dnsmasq  #centos为例
  • 配置
    vi /etc/dnsmasq.conf
  • 添加
    port=5151//使用默认53端口会被GFW拦截
no-hosts//不使用本机host
log-queries//生成日志文件
log-facility=/var/log/dnsmasq.log
  • 添加开机运行 && 启动 （如防火墙有封相应端口记得打开）
    chkconfig dnsmasq on
service dnsmasq status
service dnsmasq start
  • 搞定
    netstat -tunlp|grep 5151 #查看最终效果
    母鸡参考这篇文章

客户端配置

  • ~将53端口转发至母鸡(以 eth0 为默认网卡)~
    iptables -t nat -A OUTPUT -o eth0 -p udp --dport 53 -j DNAT --to {ip:port}

  • 检查是否实现

    dig {网站}

  • 更好的客户机实现(Update!)(需要dnsmasq中国加速)

    1、客户机同样安装上 dnsmasq,以及dnsmasq中国加速（随便叫的名字）
2、vi /etc/dnsmasq.conf:
        no-resolv
        no-poll    #让 dnsmasq 不要通过 /etc/resolv.conf 确定上游服务器
        server={母鸡ip#端口}      这里要用#引出端口
        strict-order
        conf-dir=/etc/dnsmasq.d   #dnsmasq中国加速需要
3、vi /etc/resolv.conf(第一行修改):
        nameserver 127.0.0.1
3、（重新）启动dnsmasq，如果搭了VPN服务端也重启下。
        dig {网站}            #测试

    update的方法参考这篇文章
